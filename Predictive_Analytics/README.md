## Objective
Implement and evaluate predictive modeling pipelines that balance performance, interpretability, and methodological rigor.

## Contents
- `wederstrandt_wk10_ex.ipynb`: Week 10 predictive analytics exercise notebook.
- `rm.md`: Supplemental notes.

## Technical Focus
- Feature preparation and modeling workflow implementation.
- Baseline and comparative model evaluation.
- Interpretation of predictive performance in an applied setting.

## Tooling
- Python 3.x
- Jupyter Notebook
- Typical stack: `pandas`, `scikit-learn`, visualization libraries

## Modeling Workflow
1. Prepare and validate input features.
2. Define train/test or cross-validation strategy.
3. Train baseline and candidate models.
4. Evaluate with task-appropriate metrics.
5. Compare results and document trade-offs.

## Evaluation Considerations
- Distinguish between model fit and generalization quality.
- Track class balance or target distribution impacts.
- Interpret errors to identify feature or preprocessing gaps.

## Reproducibility Notes
- Execute notebook cells in order.
- Keep dependency versions stable when comparing results across runs.
- Use supplemental notes to capture assumptions and experiment context.

## Reviewer Checklist
- Baseline vs candidate models are clearly separated.
- Metrics align with the problem type.
- Trade-offs are discussed, not only scores reported.
